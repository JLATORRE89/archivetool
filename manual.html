Introduction

▶

Website Archiver is a powerful tool for creating offline copies of websites. It supports both graphical and command-line interfaces, making it suitable for various use cases from simple one-off archives to automated batch processing.

Key Features

  • Complete website downloading with asset preservation
  • Multi-threaded architecture for faster downloads
  • Detailed logging system
  • Progress tracking and status updates
  • Container support for easy deployment
  • Both GUI and CLI interfaces
  • Configurable output directory structure
  • Dynamic content capture (AJAX/JavaScript)
  • Image compression and optimization
  • Resource inlining (CSS, JavaScript)

Installation

▶

Container Installation (Recommended)

git clone https://github.com/JLATORRE89/website-archiver
cd website-archiver
python deployarchiver.py

Manual Installation

git clone https://github.com/JLATORRE89/website-archiver
cd website-archiver
pip install -r archiver/requirements.txt

System Requirements

  • Python 3.8 or higher
  • For GUI: Tkinter support
  • For container: Podman or Docker
Note: If you're using the GUI version, ensure your system has Tkinter installed. On Ubuntu/Debian, you can install it with: sudo apt-get install python3-tk

CLI Usage

▶

Basic Usage

python -m archiver.cli https://example.com

Command Line Options

Option Description Default
-o, --output Output directory path ~/website_archives
-t, --threads Number of download threads 5
-q, --quiet Suppress progress output False
--verify-ssl Enable SSL verification True
--no-verify-ssl Disable SSL verification False

Example Commands

# Archive with custom output directory
python -m archiver.cli https://example.com -o /path/to/output

# Use 10 download threads
python -m archiver.cli https://example.com -t 10

# Quiet mode with no progress output
python -m archiver.cli https://example.com -q

GUI Usage

▶

Starting the GUI

python -m archiver.gui

Interface Components

  • URL Input: Enter the website URL to archive
  • Output Directory: Choose where to save the archive
  • Progress Bar: Shows download progress
  • Status Display: Shows current activity and file being processed
  • Control Buttons: Start and stop the archiving process
Tip: You can stop the archiving process at any time by clicking the Stop button. The partial archive will be preserved.

Container Usage

▶

Deploying the Container

python deployarchiver.py

Advanced Deployment Options

python deployarchiver.py --port 8080 --data-dir /path/to/data --rebuild --clean

CLI Mode

podman exec website-archiver python -m archiver.cli https://example.com -o /data/archive

GUI Mode (X11 forwarding required)

podman exec -e DISPLAY=$DISPLAY \
    -v /tmp/.X11-unix:/tmp/.X11-unix \
    website-archiver python -m archiver.gui

Container Volumes

  • /data: Main output directory
  • /data/logs: Log files location
Note: For GUI mode, ensure X11 forwarding is properly configured on your system.

Deploying Archived Sites

▶

NGINX Container Deployment

You can easily serve archived websites using the included NGINX container. Follow these steps:

1. Archive a Website

# Archive your website
python -m archiver.cli https://example.com -o ./data/archive

2. Deploy the NGINX Viewer

# Navigate to the nginx directory
cd nginx

# Run the deployment script
python deploy.py

# Access your site
# Open http://localhost:8080 in your browser

3. Advanced NGINX Configuration

You can customize the NGINX deployment by editing the following files:

  • nginx/nginx.conf - NGINX server configuration
  • nginx/Containerfile - Container definition
  • nginx/deploy.py - Deployment script options
Note: The default configuration serves sites on port 8080. You can change this in the deployment script.

4. Container Management

# List running containers
podman ps

# Stop NGINX container
podman stop archived-site-viewer

# Remove NGINX container
podman rm archived-site-viewer

# View logs
podman logs archived-site-viewer
Security Note: When deploying to production:
  • Use proper SSL certificates
  • Configure appropriate security headers
  • Remove any sensitive information from archived content
  • Follow NGINX security best practices

Troubleshooting

▶

Common Issues

Problem Solution
SSL Certificate Errors Use --no-verify-ssl option
Missing Assets Check website's robots.txt and JavaScript-loaded content
Slow Downloads Increase thread count with -t option
Container already exists Use --clean flag with deployarchiver.py
Permission issues Check Podman installation and permissions
Dynamic content not captured Ensure Selenium and ChromeDriver are properly installed
Selenium errors Install Chrome browser and required dependencies

Selenium & Dynamic Content Requirements

To enable dynamic content capture (AJAX/JavaScript), the following are required:

  • Chrome browser installed
  • Selenium package: pip install selenium
  • WebDriver manager: pip install webdriver-manager
Note: The archiver will fall back to static capture mode if Selenium is not available.

Log File Analysis

Check the log files in the output directory:

  • logs/archiver.log - Detailed logging information
  • logs/error.log - Error messages only

Advanced Usage

▶

Dynamic Content Capture

The archiver can capture JavaScript-generated content:

  • AJAX requests are automatically intercepted
  • JavaScript execution is simulated with Selenium
  • Dynamic elements are captured in their final rendered state

Image Optimization

Images are automatically optimized:

  • Images larger than the configured size limit are compressed
  • Quality is adjusted while maintaining visual fidelity
  • Base64 encoding is used to inline images in HTML

Performance Tuning

Tips for optimizing download performance:

  • Adjust thread count based on your system capabilities
  • Use quiet mode (-q) to reduce CPU usage
  • Consider using an SSD for the output directory
