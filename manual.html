Introduction

▶

Website Archiver is a powerful tool for creating offline copies of websites. It supports both graphical and command-line interfaces, making it suitable for various use cases from simple one-off archives to automated batch processing.

Key Features

  • Complete website downloading with asset preservation
  • Multi-threaded architecture for faster downloads
  • Detailed logging system
  • Progress tracking and status updates
  • Container support for easy deployment
  • Both GUI and CLI interfaces
  • Configurable output directory structure

Installation

▶

Container Installation (Recommended)

git clone https://github.com/JLATORRE89/website-archiver
cd website-archiver
podman build -t website-archiver .

Manual Installation

git clone https://github.com/JLATORRE89/website-archiver
cd website-archiver
pip install -r requirements.txt

System Requirements

  • Python 3.8 or higher
  • For GUI: Tkinter support
  • For container: Podman or Docker
Note: If you're using the GUI version, ensure your system has Tkinter installed. On Ubuntu/Debian, you can install it with: sudo apt-get install python3-tk

CLI Usage

▶

Basic Usage

python -m archiver.cli https://example.com

Command Line Options

Option Description Default
-o, --output Output directory path ~/website_archives
-t, --threads Number of download threads 5
-q, --quiet Suppress progress output False
--verify-ssl Enable SSL verification True

Example Commands

# Archive with custom output directory
python -m archiver.cli https://example.com -o /path/to/output

# Use 10 download threads
python -m archiver.cli https://example.com -t 10

# Quiet mode with no progress output
python -m archiver.cli https://example.com -q

GUI Usage

▶

Starting the GUI

python -m archiver.gui

Interface Components

  • URL Input: Enter the website URL to archive
  • Output Directory: Choose where to save the archive
  • Progress Bar: Shows download progress
  • Status Display: Shows current activity and file being processed
  • Control Buttons: Start and stop the archiving process
Tip: You can stop the archiving process at any time by clicking the Stop button. The partial archive will be preserved.

Container Usage

▶

CLI Mode

podman run -v /path/to/output:/data website-archiver cli https://example.com

GUI Mode

podman run -e DISPLAY=$DISPLAY \
    -v /tmp/.X11-unix:/tmp/.X11-unix \
    -v /path/to/output:/data \
    website-archiver gui

Container Volumes

  • /data: Main output directory
  • /data/logs: Log files location
Note: For GUI mode, ensure X11 forwarding is properly configured on your system.

Configuration

output_directory/

Logging Configuration

Logs are stored in the logs directory and contain:

  • Download status for each page
  • Error messages and warnings
  • Processing information for assets
  • Overall archive statistics

Troubleshooting

▶

Common Issues

Problem Solution
SSL Certificate Errors Use --no-verify-ssl option
Missing Assets Check website's robots.txt and JavaScript-loaded content
Slow Downloads Increase thread count with -t option

Log File Analysis

Check the log file at output_directory/logs/archiver.log for detailed error messages and debugging information.

Deploying Archived Sites

▶

NGINX Container Deployment

You can easily serve archived websites using NGINX in a container. Follow these steps:

1. Prepare the Archive

    # Archive your website
    python -m archiver.cli https://example.com -o ./archive

    # Verify the archive structure
    ls archive/
    # Should see: index.html, assets/, etc.

2. Create Deployment Files

Create these files in your deployment directory:

Containerfile.nginx:
FROM nginx:stable-alpine

    WORKDIR /usr/share/nginx/html
    COPY archive/ .
    COPY nginx.conf /etc/nginx/conf.d/default.conf

    EXPOSE 80

    CMD ["nginx", "-g", "daemon off;"]
nginx.conf:
server {
        listen 80;
        server_name localhost;
        root /usr/share/nginx/html;
        index index.html;

        gzip on;
        gzip_types text/plain text/html text/css application/javascript;
        
        location / {
            try_files $uri $uri/ /index.html;
            expires 30d;
        }
    }

3. Build and Run the Container

    # Build the container
    podman build -f Containerfile.nginx -t archived-site

    # Run the container
    podman run -d -p 8080:80 archived-site

    # Access your site
    # Open http://localhost:8080 in your browser

4. Optional: Configure SSL

To enable HTTPS:

    # Generate self-signed certificate
    openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
        -keyout nginx-selfsigned.key \
        -out nginx-selfsigned.crt

    # Update nginx.conf to include SSL
    server {
        listen 443 ssl;
        ssl_certificate /etc/nginx/ssl/nginx-selfsigned.crt;
        ssl_certificate_key /etc/nginx/ssl/nginx-selfsigned.key;
        ...
    }
Note: For production use, replace self-signed certificates with proper SSL certificates.

5. Additional Configuration Options

Configuration Description Example
Custom Domain Set custom server name server_name example.com;
Cache Control Configure browser caching add_header Cache-Control "public, max-age=2592000";
Compression Enable/configure Gzip gzip_comp_level 6;

6. Container Management

    # List running containers
    podman ps

    # Stop container
    podman stop CONTAINER_ID

    # Remove container
    podman rm CONTAINER_ID

    # View logs
    podman logs CONTAINER_ID
Security Note: When deploying to production:
  • Use proper SSL certificates
  • Configure appropriate security headers
  • Remove any sensitive information from archived content
  • Follow NGINX security best practices

Advanced Usage

▶

Custom Asset Handling

The archiver automatically handles common asset types:

  • Images (jpg, png, gif, svg)
  • Stylesheets (css)
  • JavaScript files (js)
  • Web fonts (woff, woff2, ttf)

Performance Tuning

Tips for optimizing download performance:

  • Adjust thread count based on your system capabilities
  • Use quiet mode (-q) to reduce CPU usage
  • Consider using an SSD for the output directory
